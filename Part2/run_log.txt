TASK 2 — Encode categorical columns
================================================================================

education_level ordinal mapping: {'High School': 0, 'Bachelors': 1, 'Masters': 2, 'PhD': 3}
Justification: education level has a genuine real-world ordering (each level represents strictly more education than the last), so mapping to increasing integers preserves meaningful information a linear model can use (e.g. a monotonic relationship with salary).

department categories: ['Engineering', 'Finance', 'HR', 'Marketing', 'Sales', 'Support']
region categories: ['Central', 'East', 'North', 'South', 'West']
Justification: department and region have no inherent ranking (e.g. 'Engineering' is not numerically greater than 'Sales'). Label-encoding them would impose a false ordinal relationship the model would wrongly treat as meaningful (e.g. implying HR < Engineering < Finance). One-hot encoding avoids this by giving each category its own independent binary column.

TASK 3 — Leak-free train-test split and scaling

Train shape: (1600, 16), Test shape: (400, 16)

IMPORTANT: the scaler was fit only on X_train, then used to transform both X_train and X_test. Fitting it on the full dataset (train+test combined) would be data leakage: the scaler's mean/std would then partly reflect the test set's own statistics, meaning information about the test set would leak into how the training features are represented, giving an optimistic bias to every metric computed later. The test set must stay statistically 'unseen' until evaluation.
